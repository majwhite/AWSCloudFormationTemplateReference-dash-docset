AWS::KinesisAnalyticsV2::ApplicationCloudWatchLoggingOption

The AWS::KinesisAnalyticsV2::ApplicationCloudWatchLoggingOption resource to add an Amazon CloudWatch log stream to monitor application configuration errors. For more information, see CloudWatchLoggingOption in the Amazon Kinesis Data Analytics Developer Guide.

Topics

Syntax

To declare this entity in your AWS CloudFormation template, use the following syntax:

JSON

{
  "Type" : "AWS::KinesisAnalyticsV2::ApplicationCloudWatchLoggingOption",
  "Properties" : {
    "ApplicationName" : String,
    "CloudWatchLoggingOption" : CloudWatchLoggingOption
  }
}

YAML

Type: "AWS::KinesisAnalyticsV2::ApplicationCloudWatchLoggingOption"
Properties:
  ApplicationName: String
  CloudWatchLoggingOption: CloudWatchLoggingOption

Properties

ApplicationName

The name of your application (for example, sample-app).

Required: Yes

Type: String

Update requires: Replacement

CloudWatchLoggingOption

Use this parameter to configure an Amazon CloudWatch log stream to monitor application configuration errors.

Required: Yes

Type: CloudWatchLoggingOption

Update requires: No interruption

Examples

ApplicationCloudWatchLoggingOption example

The following example demonstrates how to create an ApplicationCloudWatchLoggingOption resource.

YAML

BasicApplicationV2CloudWatchLoggingOption:
  Type: "AWS::KinesisAnalyticsV2::ApplicationCloudWatchLoggingOption"
  Properties:
    ApplicationName: !Ref BasicApplication
    CloudWatchLoggingOption:
      LogStreamARN:
        !Join
        - ':'
        - - 'arn:aws:logs'
          - !Ref AWS::Region
          - !Ref AWS::AccountId
          - 'log-group'
          - !Ref TestCWLogGroup
          - 'log-stream'
          - !Ref TestCWLogStream
