AWS::EFS::FileSystem LifecyclePolicy

Describes a policy used by EFS lifecycle management to transition files to the Infrequent Access (IA) storage class. For more information, see EFS Lifecycle Management in the Amazon EFS User Guide.

Syntax

To declare this entity in your AWS CloudFormation template, use the following syntax:

JSON

{
  "TransitionToIA" : String
}

YAML

  TransitionToIA: String

Properties

TransitionToIA

A value that describes the period of time that a file is not accessed, after which it transitions to the IA storage class. Metadata operations such as listing the contents of a directory don't count as file access events.

Required: Yes

Type: String

Allowed values: AFTER_14_DAYS | AFTER_30_DAYS | AFTER_60_DAYS | AFTER_7_DAYS | AFTER_90_DAYS

Update requires: No interruption
