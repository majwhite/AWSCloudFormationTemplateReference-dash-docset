AWS::EC2::NetworkInsightsAnalysis AnalysisAclRule

Describes a network access control (ACL) rule.

Syntax

To declare this entity in your AWS CloudFormation template, use the following syntax:

JSON

{
  "Cidr" : String,
  "Egress" : Boolean,
  "PortRange" : PortRange,
  "Protocol" : String,
  "RuleAction" : String,
  "RuleNumber" : Integer
}

YAML

  Cidr: String
  Egress: Boolean
  PortRange: 
    PortRange
  Protocol: String
  RuleAction: String
  RuleNumber: Integer

Properties

Cidr

The IPv4 address range, in CIDR notation.

Required: No

Type: String

Update requires: No interruption

Egress

Indicates whether the rule is an outbound rule.

Required: No

Type: Boolean

Update requires: No interruption

PortRange

The range of ports.

Required: No

Type: PortRange

Update requires: No interruption

Protocol

The protocol.

Required: No

Type: String

Update requires: No interruption

RuleAction

Indicates whether to allow or deny traffic that matches the rule.

Required: No

Type: String

Update requires: No interruption

RuleNumber

The rule number.

Required: No

Type: Integer

Update requires: No interruption
